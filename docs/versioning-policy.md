# Versioning Policy

REP-01 versions identify protocol semantics, not business profile content changes.

## Protocol Version

`REP-01` identifies this reference model and its required public projection semantics.

Breaking changes to required fields, exposure classes, authority semantics, or conformance requirements SHOULD result in a new protocol version.

## Profile Version

Profile versioning is implementation-specific. A profile MAY change without changing the protocol version.

## Reaffirmation Date

`reaffirmed_as_of` is not a protocol version and is not a fact-change date. It represents current reaffirmation of stable truth under owner authority.
